## File Descriptions

### API Routes (`app/api/`)

**Authentication:**
- `auth/login.ts` - Handles admin login, creates JWT-like token in HTTP-only cookie
- `auth/logout.ts` - Clears admin session cookie

**Reviews:**
- `reviews/route.ts` - POST endpoint to submit feedback with all protection checks:
  - Turnstile verification
  - IP-based rate limiting (3 per hour)
  - Project validation
  - Input validation

**Admin Operations:**
- `admin/projects/route.ts` - POST to create new projects
- `admin/projects/[id]/route.ts` - PUT to update, DELETE to remove projects
- `admin/reviews/[id]/route.ts` - DELETE to remove reviews

### Pages (`app/`)

**Public:**
- `page.tsx` - Home page with links to feedback and admin
- `feedback/page.tsx` - Feedback form page (validates project exists)
- `not-found.tsx` - 404 page

**Admin (Protected by middleware):**
- `admin/page.tsx` - Dashboard server component (fetches data)
- `admin/login/page.tsx` - Login form

### Components

**Client Components (Interactive):**
- `feedback/FeedbackForm.tsx` - Feedback form with all protections:
  - Honeypot field (hidden)
  - Form submission timer (rejects < 3 seconds)
  - Turnstile CAPTCHA
  - Star rating
  - Comment textarea
- `feedback/Turnstile.tsx` - Cloudflare Turnstile widget
- `admin/AdminDashboard.tsx` - Tab switching between projects and reviews
- `admin/ProjectManagement.tsx` - Project CRUD UI
- `admin/ReviewFeed.tsx` - Review list with filtering and deletion

### Utilities (`lib/`)

- `supabase.ts` - Creates Supabase admin client (service role key)
- `security.ts` - Security functions:
  - `hashIP()` - SHA-256 hash IP addresses
  - `getClientIP()` - Extract client IP from headers
  - `verifyTurnstile()` - Verify Turnstile token with Cloudflare
  - `generateSecretKey()` - Generate random secrets

### Configuration Files

- `middleware.ts` - Protects `/admin` routes, checks session cookie
- `next.config.ts` - Next.js optimization and security headers
- `tailwind.config.ts` - Tailwind CSS theme customization
- `postcss.config.js` - PostCSS setup for Tailwind
- `tsconfig.json` - TypeScript strict mode enabled
- `package.json` - Dependencies and scripts

### Database (`DATABASE_SCHEMA.sql`)

Creates:
- `projects` table with slug as unique identifier
- `reviews` table with foreign key to projects
- Indexes for performance optimization
- Row Level Security (RLS) policies
- Example seed data

### Documentation

- `README.md` - Complete feature documentation, usage, troubleshooting
- `QUICKSTART.md` - 5-minute setup guide for local development

## Key Features by Location

### Bot Protection Layers

1. **Honeypot Field** (`FeedbackForm.tsx`):
   - Hidden CSS field that bots typically fill
   - Silent rejection if populated

2. **Form Timer** (`FeedbackForm.tsx`):
   - Tracks page load time
   - Rejects submissions < 3 seconds

3. **Turnstile CAPTCHA** (`Turnstile.tsx` + `reviews/route.ts`):
   - Client-side widget rendering
   - Server-side verification with secret key

4. **IP Rate Limiting** (`reviews/route.ts`):
   - Hashes IP address (SHA-256)
   - Checks max 3 reviews per IP per hour in Supabase

### Security

- **Middleware** (`middleware.ts`):
  - Protects all `/admin/*` routes
  - Validates session token on every request
  - Redirects to login if invalid

- **API Routes**:
  - Service role key never exposed to client
  - All sensitive operations server-only
  - Input validation on all endpoints
  - Error handling without information leakage

- **Cookies**:
  - HTTP-only flag prevents JavaScript access
  - Secure flag for HTTPS only (production)
  - SameSite strict prevents CSRF
  - 24-hour expiration

## Data Flow

```
Public User Journey:
1. User visits /feedback?project=app-one
2. page.tsx verifies project exists
3. FeedbackForm renders with protections
4. User submits form
5. Turnstile verifies -> FeedbackForm checks timer/honeypot
6. Sends POST to /api/reviews with token
7. API route verifies Turnstile, checks IP rate limit
8. Inserts review into Supabase

Admin User Journey:
1. User visits /admin/login
2. middleware.ts allows login page
3. User submits credentials
4. /api/auth/login validates, sets cookie
5. Redirect to /admin
6. middleware.ts checks cookie, allows access
7. page.tsx fetches projects/reviews from Supabase
8. AdminDashboard renders client component
9. Admin performs CRUD operations via API routes
```

## Performance Considerations

- **React Server Components**: Minimal JavaScript sent to client
- **Database Indexes**: Fast queries for filtering and rate limiting
- **Static CSS**: Tailwind minified
- **Vercel CDN**: Caches static assets globally
- **Serverless Functions**: No idle servers, scales automatically
- **No Long Queries**: Rate limit checks use indexed IP hash
